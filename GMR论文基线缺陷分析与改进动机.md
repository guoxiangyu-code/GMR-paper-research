# 揭开数字的遮羞布：GMR 基线模型的核心缺陷与评测悖论

在对 Soccer-GMR 论文（Retrieving Any Relevant Moments）的官方代码及复现结果进行深度诊断与鲁棒性验证后，我们发现：**论文主推的 GMR Adapter 拒识能力，本质上是一个“未经分数校准、靠固定阈值侥幸对齐”的脆弱机制。** 
论文所汇报的高分（如 Rej-F1）高度依赖于阈值恰好卡在特定 Checkpoint 难样本簇的边缘；而模型在 GMR 真正的核心难点（多时刻正例 vs 语义相近的空集）上几乎丧失了判别力。这一点被宏观的 AUROC 指标和“贴阈值的及格”巧妙地掩盖了。

以下我们将从四个结构性维度，系统剖析该论文在方法、评测与可复现性上的深层缺陷，这也为后续的优化工作提供了最坚实的立论基础。

---

### 一、 方法贡献的“成色”被高估：未经校准的脆弱门控

论文将 GMR Adapter 包装为两大方法贡献之一，主打“轻量即插即用地赋予判别式模型拒识能力”，其核心证据是 Rej-F1 从 0 跃升至 64。然而，诊断表明这一跃升的**稳定性极差**。

Adapter 的存在性头（Existence Head）仅通过 Max-Pooling 和两层 MLP 构成，采用无类别加权的 BCE 损失进行训练。在训练配置中，该分支既缺乏分数校准约束，其损失权重（1.0）又被主任务（时序边界 10.0、分类 4.0）严重压制。
这导致它学到的**绝对分数（Logit Bias）是放任自流的**。只要训练的随机种子或早停时机稍有变动，预测分数的整体分布就会在数轴上发生剧烈漂移（如我们复现时整体右移了 +0.13）。
因此，论文中 Rej-F1 达到 64.01 的优异表现，**并不是“模型真正学会了稳健拒识”的证据，而仅仅是“默认阈值 $\tau=0.4$ 恰好命中了官方那一次训练分布的及格线”的巧合产物。** 当同样的阈值施加在分布漂移后的复现模型上时，拒识能力瞬间归零。

### 二、 评测协议与任务目标的内在悖论

这是论文在方法论上最致命的结构性问题。论文反复强调 GMR 的精髓是“拒识 + 定位”的联合能力，并为此专门设计了 G-mIoU 端到端指标。但在其官方训练与评测管线中，却在两个关键环节背叛了这一目标：

1. **对负样本盲视的验证集**：在验证集（Validation Set）构建逻辑中，代码硬性设定了 `keep_empty_gt=False`，**强行过滤掉了所有的空集（负）样本**。
2. **片面的模型选择准则**：在挑选最优 Checkpoint 时，代码仅依赖 `MR-full-mAP`（只评估正样本的定位能力）。

这意味着，**模型选择的整个闭环，从未为“拒识误报（False Positives）”付出过任何代价**。一个把拒识作为核心挑战和最大卖点的工作，却用一套对拒识能力完全盲视的准则来挑选最佳模型，存在严重的逻辑割裂。这进一步加剧了存在性得分不可控的乱象。

### 三、 指标掩盖了任务真相，暴露了两大挑战的零和博弈

把官方预测的分数分布摊开来看，我们发现了论文未曾揭示的最有价值的真相：

| 论文的宣称 / 呈现 | 数据诊断揭示的真相 |
| :--- | :--- |
| AUROC 高达 70~74，拒识“判别力良好” | 这个分数几乎全靠极其简单的“单时刻（Single）样本”（得分极高，中位数 $\approx 1.0$）撑起。 |
| 多时刻与拒识是两个并列的新挑战 | 事实是：二者被强行绑定在同一个一维闸门上，存在直接的**零和冲突**。 |
| $\tau=0.4$ 下 Rej-F1 及格，拒识“有效” | 官方数据中，**多时刻正例的中位数为 0.395 < 0.4**。同一道阈值在拦截负样本的同时，**粗暴地错拒了超过半数的多时刻正样本**。 |

真正的危机在于：在 GMR 最该解决的那对难样本（多时刻正例 vs 语义相近的负例）上，两者的分数中位数分别聚集在 0.395 与 0.388。**间隔不足 0.01，存在性头在这里的判别能力几乎等同于抛硬币。** 
论文宣称 "multi-moment remains a key open challenge"（多时刻检索仍是核心挑战），但其将原因归咎于检索不全（mR+@5 偏低）。我们的诊断将这一结论推向了更深层：困难不仅在于框不准，更在于**基线模型连“这个多时刻查询到底算不算正样本”都无法判别**。提高阈值拦截误报，就必然扼杀多时刻召回；降低阈值保召回，就会放进海量误报。论文完全掩盖了两者共享同一门控所带来的**此消彼长（Trade-off）** 的本质矛盾。

### 四、 可复现性缺口与基准的科学严谨性

最后是科学严谨性问题。既然绝对分数是 Checkpoint 相关的，论文中大量报告的 Rej-F1、G-mIoU 等**强依赖单点固定阈值（$\tau=0.4$）的绝对数字，在他人复现时天然不具备可比性**。
任何研究者在使用不同随机种子复现时，极大概率会像我们一样，得到一个判别力（AUROC）相当、但单点指标彻底“坍塌”的结果。论文既没有报告多 seed 的方差，也没有在正文中强调“默认阈值需按验证集动态标定”的前提。这就使得主表中呈现的 Baseline 数据，更像是“某一次幸运训练的快照”，而非一个坚固、可稳定对照的基准。对于一篇立足于建立 Benchmark 和提供 Strong Baselines 的论文而言，这是一个严重的短板。

### 五、 结语与后续研究动机

综上所述，这些发现**丝毫没有否定 GMR 任务设定的价值**——相反，它从反面极大地证明了 GMR 引入拒识和多时刻挑战的必要性和迫切性。
它真正动摇的，是这篇论文所提供的**现成解法与评测管线**。这正是我们后续引入“基数感知（Cardinality-aware）与分数校准（Calibration）”模块最无懈可击的立论起点：**现有基线不仅远未解决 GMR 的核心冲突，甚至连诚实度量这种冲突都还没做到。**
