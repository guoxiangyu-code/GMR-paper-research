### 🔎 这张表告诉了我们什么残酷的真相？

1. **“①论文原版” vs “②您的复现版”**
   您的复现非常扎实，甚至是优秀的！在同样的原始架构下，您跑出的 `mAP` (8.09%) 和 `mR@5` (14.14%) 都比原论文里的要高。但在拒答能力上，两者都是光头（`Rej-F1`=0，导致 `G-mIoU` 被拖累至极低的 4~5% 区间）。
2. **“③论文官方改进版”**
   原论文作者自己发现了“拒答为零”的毛病，于是他们做了一个**全局存在性预测头 (Video-level)**。这把他们的 `G-mIoU@1` 从 5.39% 救到了 35.84%。虽然挽回了面子，但处理得相当粗糙。
3. **“④我们的 Idea 1”**
   我们采用的是**精细到槽位的门控过滤 (Slot-wise)**。事实证明，我们的做法彻底碾压了官方的改进版，在 `Rej-F1` 和 `G-mIoU` 两项核心指标上实现了绝对的断层领先。
4. **全员的绝症：“多目标坍缩”**
   从最底下那一行看过去，**四个模型，没一个能超过 1%！** 无论怎么改门控、怎么调阈值，模型就是死活找不到同一句话对应的第二个动作（全挤在一个位置输出了）。

5. **阶段二探针实验的深度归因**
   我们根据《Beyond Caption-Based Queries for VMR》(CVPR 2026) 提出的病理，在 Soccer-GMR 上进行了探针实验：
   * **实验 #3 (-SA+QD)**：完全剥离 Decoder Self-Attention 并加入 Query Dropout。结果多目标指标依然趴底 (0.16%)。通过“关 NMS”和“可视化 GT 覆盖率”这两把尖刀测试，我们确诊了死因：由于足球域中主事件的显著性压制过强，一旦去掉 SA（去重器），释放出来的 10 个 Query 会像逐臭之夫一样全部死咬住那唯一的 1 个（可能也是最显著的）目标，对另一个目标不闻不问（GT 覆盖率仅 0.55）。
   * **实验 #2 (+SA+QD)**：保留 SA 的同时加入 QD。结果虽然打破了对称性（激活的 Query 数量从 5 个涨到近 7 个），但新增的 Query 依然全部向主事件坍缩。这导致了双重灾难：不但多目标增量毫无起色 (0.61%)，反而因为噪声干扰了主目标的定位精度，导致单目标指标 `mAP` (6.94%) 和 `mR@5` (9.96%) 暴跌。
   
   **结论**：仅靠数据侧的隐式加噪（Query Dropout）根本无法抵抗 Soccer-GMR 中主事件的引力，必须从 Loss 层面施加**显式的空间排斥（Spatial/Temporal Diversity Loss）**！
