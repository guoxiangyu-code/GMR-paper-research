## 1. 配置开关(本轮只动主效应,NMS 阈值锁 0.7)

| 开关 | 值 | 说明 |
|---|---|---|
| use_sa | False | 删 decoder self-attn,治 coordination collapse |
| query_dropout | 0.25 | 锁定 0.25;0.5 会崩(论文 Table 8) |
| use_nms | True | 仅位置去重,不砍分数、不动空集判定 |

| seed | 42 | QD 带随机性,固定 42 |

> 命令:`CUDA_VISIBLE_DEVICES=0 ./scripts/train_moment_detr_gmr.sh --use_sa False --query_dropout 0.25 --use_nms True --nms_thr 0.7 --seed 42 --results_dir experiments/20260627_stage2_run1/results`

## 4. 结论与下一步

- [ ] **通过**(active 数上扬 + mR+@5 显著涨 + 守护不破)
  → 排 #1(仅 -SA)、#2(仅 QD)归因实验,凑齐 Table 7;再单独扫 nms_thr {0.5,0.6,0.7,0.8}(仅验证集)。
- [x] **失败**(mR+@5 不动 /守护破阈)
  → 不跑 #1/#2(论文已证单独无效),掉头查 Soccer-GMR 域特异性(GT≥4 仅 12 样本,统计量薄)。

**结论**: 理论预测的“治愈坍缩”在底层机制上发生了（active query 从 4.9 抬升至 9），但**业务指标完全没接住**。释放出来的额外 Query 沦为了漫无目的的瞎猜，非但没有抓到多目标（mR+@5 暴跌至 0.16%），反而制造了大量 False Positives，导致 `G-mIoU@1` 和 `Rej-F1` 双双跌破底线。
**决策**: 终止原定的消融矩阵（#1/#2 不必跑了）。立即掉头进行 Soccer-GMR 的域特异性分析。
