# 实验 #2: 保留 SA + 引入 Query Dropout (多样性注入)

## 1. 实验动机
根据前置探针实验（#3）的两刀归因测试结果：
- **关 NMS 测试**: `ΔmR+@5 = 0.0`，排除了 NMS 误删的嫌疑。
- **GT 覆盖率测试**: GT≥2 的样本平均覆盖率仅为 0.553（62.5%的样本覆盖率≤0.5）。
这实锤了在 Soccer-GMR 域中，去掉 Self-Attention 会导致严重的**“Query 塌缩到主目标”**副作用。

因此，本实验重新激活原定被放弃的 **#2 消融（仅 +QD，保留 -SA）**：
- **保留 SA**：让 Decoder Self-Attention 继续发挥去重、协调的本职工作，防止 Query 扎堆。
- **引入 Query Dropout (0.25)**：在输入端注入噪声，打破对称性，逼迫闲置的 Query 去寻找次要目标。

## 2. 核心配置
```bash
# 启动命令预演
./scripts/train_moment_detr_gmr.sh \
  --use_sa True \
  --query_dropout 0.25 \
  --bsz 2048 \
  --eval_bsz 2048 \
  --epochs 400
```
*注: NMS 属于推理期开关，由于在保留 SA 时模型自带去重，测试时暂不开启 NMS，或仅作为对比评估。*

## 3. 预期目标
1. **多目标指标**: `mR+@5` 需要突破基线 1.12%，且显著高于 #3 的 0.16%。
2. **守护指标**: `G-mIoU@1` 和 `Rej-F1` 不得破线，保持稳健。
3. **Query 坍缩**: `diagnose.py` 测得的 GT≥2 时的 active query 数应大于基线 4.9，但也不必像 #3 那样虚高到 9（因为 SA 会抑制无效的 False Positives）。

## 4. 实验结果
- mAP: 6.94 (相比基线 9.94 暴跌)
- mR@5: 9.96 (相比基线 14.93 暴跌)
- mR+@5: 0.61 (不仅未达标，甚至不如基线的 1.12)
- G-mIoU@1 (@0.6): 51.13
- Rej-F1 (@0.6): 74.73
- GT 覆盖率: 0.559（和完全剥离 SA 的 0.553 几乎一致）
- Active Queries: GT=2时约6.85，GT=4时约7.00 (比基线5高，但无效)

## 5. 结论
在 Soccer-GMR 中，主事件的“特征黑洞”效应极强。Query Dropout 确实通过注入噪声打破了对称性，逼迫模型激活了更多的 Query（从 5 增加到了 7），但这些新增的 Query 依然全部向主事件坍缩。这导致了双重灾难：
1. **多目标能力无效**：多出的 Query 依然扎堆，未能覆盖次要目标（覆盖率仍为 0.5 左右）。
2. **单目标能力受损**：噪声干扰了原本主目标的定位精度，导致基础指标 `mAP` 和 `mR@5` 大幅下滑。

结论：仅靠数据侧的隐式加噪（Query Dropout）根本无法抵抗主事件的引力，必须从 Loss 层面施加**显式的空间排斥（Spatial/Temporal Diversity Loss）**！
