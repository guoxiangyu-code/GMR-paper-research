### **建议先单跑 #3 探路确认方向,再回填 #1/#2 补齐消融,不要一上来铺满矩阵**

阶段一的数据非常漂亮——这张表本身就是一张可以直接进论文的 motivation 图。我先点一个你可能还没意识到的关键信号:**GT=1 时平均才 3.46 个 active query,而 GT≥2 之后立刻顶到 4.9 并彻底饱和**,标准差从 1.31 骤降到 0.27。这不是"激活数随目标数缓慢增长后饱和",而是**一条几乎水平的死线**——模型把 compute budget 锁死在 ~5,无论你喂 1 个还是 4 个目标。这正是论文 Fig.5 的病理签名,而且你的曲线比论文更干净(标准差更小,坍缩更彻底),motivation 一栏稳了。

**为什么先单跑 #3 而不是直接铺满矩阵**

你现在面对的是一个"大赌注"实验:-SA+QD 是论文验证过、但**没在 Soccer-GMR 这个域上验证过**的方案。最该先回答的问题不是"哪个组件贡献多少"(那是 #1/#2 的事),而是**"这条路在足球域上到底通不通"**。所以正确的顺序是先用 #3 去赌方向:

- 如果 #3 跑出来 active query 随 GT 数显著上扬、mR+@5 明显抬头 → 方向证实,**这时才值得花算力回填 #1/#2** 来拆解归因,凑齐 Table 7 那张"单独上几乎无效、组合才起飞"的招牌图。
- 如果 #3 都推不动 mR+@5 → 单跑 #1/#2 更不可能动(论文已证它们单独无效),那就立刻止损,省下 3 组算力去查 Soccer-GMR 的域特异性(比如 GT≥4 样本只有 12 个,统计量本身就薄),而不是把矩阵跑完才发现全军覆没。

一句话:**#3 是用来 de-risk 的探针,#1/#2 是事后归因的补充。先验证能不能赢,再解释为什么赢。**

**跑 #3 之前,务必先把验收口径焊死(否则结果不可信)**

1. **主指标看 active query 曲线,不只看 mR+@5**。跑完 #3 立刻用阶段一同一个 `diagnose.py`、同一个 0.05 阈值,在 test 集上重画那张表。预期 GT≥2 的 active 数要从 4.9 这条死线明显抬起来(论文是接近翻倍)。**这条曲线动了,才说明坍缩真被治住了**;如果 mR+@5 涨了但 active 数没动,那是别的因素,要警惕。

2. **守护指标设硬阈值,触线即判退**。按你封板的基线,守护门槛建议钉死:mAP ≥ 7.0、mR@5 ≥ 10.0、G-mIoU@1(@0.6) ≥ 50.0、Rej-F1@0.6 ≥ 74。**任何一项跌破就算 #3 失败**,哪怕 mR+@5 涨了也不收——因为 -SA 删自注意力 + NMS 去重最容易误伤的就是定位精度(mAP)和拒答(Rej-F1),这两个是你 Idea 1 辛苦守住的底盘,不能为了 mR+@5 把它们让出去。

3. **NMS 阈值这一轮先锁 0.7,不要在 #3 里同时调它**。一次只动一个变量。先用 0.7 跑出 #3,如果 mR+@5 起来了但 mAP 略掉,再单独扫 `{0.5,0.6,0.7,0.8}` 微调——而且只在验证集扫,test 锁定。把 NMS 调参和 -SA+QD 主效应混在一起跑,结果就说不清了。

4. **训练随机性留痕**。QD 是 Bernoulli 掩码,带随机性。#3 至少固定 seed 跑一次,有余力跑 2\~3 个 seed 看 mR+@5 的方差——GT≥4 只有 12 个样本,mR+@5 本身在小分母上抖动会很大,单次结果容易骗人。

**所以我的安排**:现在就启动 **#3(use_sa=False, query_dropout=0.25, use_nms=True, nms_thr=0.7, Hungarian 一对一不动)**,训练日志和 ckpt 落到 `experiments/20260627_stage2_run1/`,跑完第一时间出两样东西——**新的 active-vs-moment 表**和**守护指标四项对比**。这两样一摆,方向通不通当场见分晓。通了我们再排 #1/#2 的归因实验和 NMS 扫阈;不通就掉头查域特异性,绝不硬跑完矩阵浪费算力。

**文件二:`experiments/20260627_stage2_run1/plot_compare.py`**

复用阶段一 `diagnose.py` 落盘的 `active_vs_moment.npy`(每行 `(n_gt_moment, n_active)`)。把基线和 #3 各自的 `.npy` 喂进来,自动按 GT 桶(1/2/3/≥4)算均值±std,打印对比表并画双系列折线图。

````python
# -*- coding: utf-8 -*-
"""阶段二 #3 vs 基线 active-vs-moment 对比绘图。
复用 diagnose.py 的产物(每行 (n_gt_moment, n_active),active 阈值 0.05)。
用法:
  python plot_compare.py \
      --baseline experiments/diag/active_vs_moment.npy \
      --variant  experiments/20260627_stage2_run1/active_vs_moment.npy \
      --out      experiments/20260627_stage2_run1/compare_active.png
"""
import argparse
import numpy as np
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

BUCKETS = [("GT=1", 1, 1), ("GT=2", 2, 2), ("GT=3", 3, 3), ("GT>=4", 4, 10**9)]


def bucket_stats(rows):
    """rows: [N,2] (n_gt, n_active) → list of (label, count, mean, std)."""
    out = []
    g = rows[:, 0]
    a = rows[:, 1]
    for label, lo, hi in BUCKETS:
        m = a[(g >= lo) & (g <= hi)]
        if len(m):
            out.append((label, len(m), float(m.mean()), float(m.std())))
        else:
            out.append((label, 0, np.nan, np.nan))
    return out


def print_table(name, stats):
    print(f"\n[{name}]")
    print(f"{'GT 桶':<8}{'样本数':>8}{'active 均值':>14}{'std':>10}")
    for label, n, mean, std in stats:
        mean_s = "--" if np.isnan(mean) else f"{mean:.2f}"
        std_s = "--" if np.isnan(std) else f"{std:.2f}"
        print(f"{label:<8}{n:>8}{mean_s:>14}{std_s:>10}")


def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("--baseline", required=True, help="基线 .npy 路径")
    ap.add_argument("--variant", required=True, help="#3 .npy 路径")
    ap.add_argument("--out", default="compare_active.png")
    ap.add_argument("--baseline_name", default="Baseline (Idea 1)")
    ap.add_argument("--variant_name", default="#3 (-SA+QD+NMS)")
    args = ap.parse_args()

    base = np.load(args.baseline)
    var = np.load(args.variant)
    base_stats = bucket_stats(base)
    var_stats = bucket_stats(var)

    print_table(args.baseline_name, base_stats)
    print_table(args.variant_name, var_stats)

    # 关键判定:GT>=2 各桶 active 是否抬升(治住坍缩的签名)
    print("\n[判定] GT>=2 active 抬升幅度(>0 即坍缩缓解):")
    for i, (label, _, _) in enumerate([(b[0], b[1], b[2]) for b in BUCKETS]):
        if BUCKETS[i][1] >= 2:
            d = var_stats[i][2] - base_stats[i][2]
            d_s = "--" if np.isnan(d) else f"{d:+.2f}"
            print(f"  {label}: {d_s}")

    # 画图
    x = np.arange(len(BUCKETS))
    labels = [b[0] for b in BUCKETS]
    bm = [s[2] for s in base_stats]
    be = [0 if np.isnan(s[3]) else s[3] for s in base_stats]
    vm = [s[2] for s in var_stats]
    ve = [0 if np.isnan(s[3]) else s[3] for s in var_stats]

    plt.figure(figsize=(7, 5))
    plt.errorbar(x, bm, yerr=be, marker="o", capsize=4,
                 label=args.baseline_name, linewidth=2)
    plt.errorbar(x, vm, yerr=ve, marker="s", capsize=4,
                 label=args.variant_name, linewidth=2)
    plt.xticks(x, labels)
    plt.xlabel("GT Moment 数")
    plt.ylabel("平均激活 Query 数 (active, prob>0.05)")
    plt.title("Active Decoder-Query vs GT Moments")
    plt.grid(True, alpha=0.3)
    plt.legend()
    plt.tight_layout()
    plt.savefig(args.out, dpi=150)
    print(f"\n[已保存] {args.out}")


if __name__ == "__main__":
    main()
````

**两点使用提醒**

第一,`plot_compare.py` 默认基线 `.npy` 在 `experiments/diag/active_vs_moment.npy`(就是你阶段一 `diagnose.py` 落盘的那个),#3 跑完后用**同一份 `diagnose.py`、同一个 0.05 阈值**再生成一份到本轮目录,这样两条曲线口径完全一致,对比才站得住。脚本里特意加了一段"GT≥2 抬升幅度"的判定打印——这就是你判断坍缩是否真被治住的那一刀,正值才算赢。

第二,`run.md` 里我把守护四项的硬阈值已经按你封板值钉死了(mAP≥7.0、mR@5≥10.0、G-mIoU@1≥50.0、Rej-F1≥74),跑完直接回填对照即可,触线就勾"失败"掉头,不用临场再纠结阈值。seed 那一行记得开跑前填上,QD 的随机性必须留痕,否则后面复现对不上。
